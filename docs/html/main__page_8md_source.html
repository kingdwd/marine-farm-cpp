16 ## Overview
17 <a name="Overview"></a>
18 
19 This projects tackles informative path planning for an underwater robot surveying an algae farm. It proposes a simulation of the algae farm, the robot and its sensors, along with a navigation pipeline collecting information about the farm, generating trajectories to collect as much information as possible and controlling the robot to follow it. The entire architecture of the software can be found in the following figure:
20 
21 ![](images/full_architecture.svg)
22 
23 It is composed of several modules, split into a simulation and a navigation pipeline. The farm simulator generates algae lines and populates it with algae. Each algae has an interest heatmap which encodes zones of interest that need to be surveyed. The robot is simulated by integrating its dynamic equations at a regular pace, and its camera by casting rays and checking for collisions with a collision engine. A Gaussian Process updated by Kalman Filter gathers information about the farm. Using this, a path planner samples viewpoints along the algae wall and selects the ones that maximises information gain. Finally, a Model Predictive Controller generates control input to send to the robot so that it follows the desired reference trajectory while taking dynamic constraints into account. A high level planning module handles switching between algae lines or between wall sides for example.

38 ## Getting started
39 <a name="Getting-started"></a>
40 
41 This software requires <a href="https://www.ros.org/">ROS</a>. It has been tested on Ubuntu 16.04 with ROS Kinetic. To install and start using it, simply follow these steps:
42 ```
43 mkdir -p ros_ws/src
44 cd ros_ws/src
45 git clone https://github.com/CorentinChauvin/marine-farm-cpp.git
46 catkin build
47 source ../devel/setup.bash
48 
49 roslaunch mf_experiments one_wall.launch
50 ```
51 
52 To generate Doxygen documentation, issue:
53 ```
54 cd ros_ws/src/marine-farm-cpp/docs
55 doxygen
56 ```

59 ## Main launchfile
60 <a name="Main-launchfile"></a>
61 
62 `mf_experiments/launch/one_wall.launch` is the main launchfile to start a simulated experiment where the robot surveys one algae wall. It has allows to select options in the introduction of the file. The different options are the following:
63 
64 Parameter name | Meaning
65 -------------- | -------
66 `rviz` | Whether to run Rviz visualisation
67 `display_img_gp` | Whether to visualise the map estimated by Gaussian Process
68 `display_cov_gp` | Whether to visualise the Gaussian Process covariance
69 `display_diff_gp` | Whether to visualise the difference between Gaussian Process map and ground truth
70 `path_planning` | Whether to start the path planner
71 `back_forth_traj` | Whether to publish a predefined back and forth path
72 `planner_initially_disabled`| Whether the planner is disabled when surveying the first side of the wall. If so, the robot will follow the back and forth trajectory if `back_forth_traj` is set to `true`
73 `mpc_control` | Whether to start the Model Predictive Controller
74 `cart_control` | Whether to start a cartesian controller. If so, the robot will follow the reference trajectory without taking care of any dynamic constraint.
75 `accelerate_time` | Whether to change the time factor of the simulation, to accelerate the process
76 `experiment_stats` | Whether to start the process recording statistics about the experiment
77 `save_gp_mean` | Whether to save the mean of the Gaussian Process in a file at the end of the experiment
78 `load_gp_mean` | Whether to load the mean of the Gaussian Process from a file at the beginning of the experiment

81 ## Configuration files
82 <a name="Configuration-files"></a>
83 
84 The different modules of the pipeline can be configured through several configuration files:
85 
86 File name | Configuration
87 --------- | -------------
88 `mf_farm_simulator/config/farm_simulator.yaml` | Farm generation
89 `mf_robot_simulator/config/robot_simulator.yaml` | Robot model and simulation
90 `mf_sensors_simulator/config/sensors_simulator.yaml` | Camera simulation
91 `mf_mapping/config/mapping.yaml` | Gaussian Process mapping
92 `mf_navigation/config/navigation.yaml` | Path planning and control
93 `mf_experiments/config/one_wall.yaml` | Details about the algae wall surveying experiment, and overwrite some specific parameters defined previously
